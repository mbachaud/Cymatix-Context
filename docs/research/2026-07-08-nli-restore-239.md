4. **The public model is the right instrument for delivery-coherence.** On the same smoke pairs, `cross-encoder/nli-deberta-v3-small` (SNLI/MNLI, already in the HF cache) correctly flags the conflicting-delivery case as **contradiction (1.00)** where the genome head says independence (0.999).

## Why the genome-trained head is degenerate

`training/export_nli_data.py` builds pairs three ways: a **heuristic** over gene
structure (domain/entity Jaccard + co-activation), an optional **Ollama teacher**,
and an optional codon strategy. The heuristic emits only `{entailment,
reverse_entailment, equivalence, alternation, independence}` — `negation`/`cover`
come *only* from the teacher.

On the current **756-gene** genome (the README assumes 3,500–7,300), the corpus
has almost no genuine entailment/contradiction gene-pairs, so both label sources
concentrate on `independence`:

| label | count | share |
|---|---:|---:|
| entailment | 3 | 0.1% |
| reverse_entailment | 1 | 0.0% |
| equivalence | 34 | 0.8% |
| alternation | 3 | 0.1% |
| **negation** | **0** | **0.0%** |
| cover | 6 | 0.1% |
| **independence** | **4453** | **99.0%** |

3000 heuristic pairs were **100% `independence`**; even the 1500 teacher pairs came
back overwhelmingly `independence`. Inverse-frequency class weighting cannot
manufacture a decision boundary from 0–34 examples of a class, so the head learns
"predict `independence`."

### Smoke test (dispositive)

| pair | genome head (7-class) | public `nli-deberta-v3-small` (3-class) |
|---|---|---|
| "Redis is an in-memory cache backend." ↔ "The cache uses Redis with a TTL." | INDEPENDENCE 0.999 | neutral 1.00 |
| **"listens on port 11437." ↔ "listens on port 8080."** | **INDEPENDENCE 0.999** | **contradiction 1.00** |
| "Ingestion runs on CPU." ↔ "unrelated statement about fruit." | INDEPENDENCE 0.999 | contradiction 1.00* |

\* the public model over-calls the last (unrelated) pair as contradiction — it is
not perfect — but the **load-bearing case for delivery-coherence** (detecting two
*conflicting* delivered spans) is the port conflict, which it gets right and the
genome head misses entirely.

---

## Recommendation

1. **Do not use a genome-heuristic-trained NLI head for delivery-coherence.** It
   was designed to *type co-activation links* between documents by structural
   overlap, not to detect semantic contradiction between arbitrary delivered
   spans, and on this corpus it is degenerate regardless.
2. **Use the cached public `cross-encoder/nli-deberta-v3-small`** (SNLI/MNLI) as
   the delivery-coherence instrument. It ships correct contradiction/entailment/
   neutral semantics with no training and already lives in the HF cache. Its
   3-class output maps cleanly onto the coherence sign needed for "do these two
   delivered spans conflict?".
3. **If the 7-class MacCartney head is ever wanted** (for typed co-activation
   edges, its original purpose), it needs either (a) a much larger, more diverse
   genome, or (b) a real NL-relation training corpus — not genome gene-pair
   heuristics. Track under a separate issue; it is not on the #239 path.
4. **Artifact hazard:** the degenerate model now sits at `training/models/nli/`
   (the code-default `nli_model_path`), 567 MB, gitignored. It is inert under the
   shipped config, but if an operator sets `backend="deberta"` it will load and
   emit plausible-looking but garbage coherence (everything `independence`) —
   *worse* than absent, which safely soft-fails to an empty graph. Recommend
   deleting it (or moving it off the default path) unless kept deliberately as
   evidence. Left in place pending your call.
