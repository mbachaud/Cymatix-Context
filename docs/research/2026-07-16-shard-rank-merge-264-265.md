# Sharded rank-merge comparability fixes — #264 / #265 receipt

**Date:** 2026-07-16 · **Branch:** `fix/264-265-shard-rank-merge` · **Umbrella:** #275

Both defects share one root cause (#275, "rank-merge comparability"): the
cross-shard merge in `ShardRouter.query_genes` applies **additive/BM25-scale**
arithmetic to per-shard scores that are now **RRF-scale**, because production
per-shard `Genome`s default to `fusion_mode="rrf"` (KnowledgeStore default;
fanned via `open_read_source → ShardedGenomeAdapter → ShardRouter →
Genome(path, **kwargs)`).

## Fix shapes

* **#264 — doc-type boost** (`shard_router.py`, `DOC_TYPE_BOOST=1.15`, the
  `corrected *= boost` block). New DEFAULT-INERT knob
  `[retrieval] doc_type_boost_mode = "additive" | "rank" | "off"`:
  * `additive` (**default**) — the shipped ×1.15 magnitude multiply,
    byte-for-byte.
  * `off` — no lift. The RRF-honest escape; satisfies #121's hard constraint.
  * `rank` — the lift becomes a rank-domain **tier fed into the cross-shard
    Fuser** (issue candidate b); the final merge keys primarily on the
    rank-fused score. `corrected` is left unmultiplied.
* **#265 — global-IDF splice** (`shard_router.py`, the `HELIX_SHARD_GLOBAL_IDF`
  block `corrected = raw − old_local_lex + new_global_lex`). **Capability
  guard, not a knob**: the splice mixes BM25-magnitude (0–6, `FTS5_LEXICAL_CAP`)
  lexical sub-scores into `raw`, which is only scale-coherent when `raw` is
  additive-scale. Under per-shard RRF (`raw ≈ Σ 1/(k+rank) ≈ 0.0x`) it is
  undefined, so it is hard-gated to additive-mode shards and logs a
  once-per-process suppression notice under RRF (`_warn_global_idf_rrf_suppressed`).

Default byte-identity: `doc_type_boost_mode="additive"` + the additive-only
guard change nothing on the shipped default path (existing shard-router /
global-idf / score-debug suites pass unchanged).

### #264 — README vs keyword-dense impl (`fusion` × `doc_type_boost_mode`)

| cell | impl rank | README rank | flip (impl < README)? | impl | README |
|---|---|---|---|---|---|
| **rrf / additive (default)** | 1 | 0 | **YES (defect)** | 3.0376 | 3.4913 |
| rrf / off | **0** | 1 | no ✅ | 3.0376 | 3.0359 |
| rrf / rank | 1 | 0 | YES | 3.0376 | 3.0359 |
| additive / additive | 0 | 1 | no | 13.9155 | 9.6859 |
| additive / off | 0 | 2 | no | 13.9155 | 8.4225 |
| additive / rank | 1 | 0 | YES | 13.9155 | 8.4225 |

Reading: under per-shard RRF the default ×1.15 flips the keyword-dense impl
below the README (3.4913/3.0376 = 1.149 ≈ the boost). **`off` is the mode that
satisfies #121's hard constraint** ("never flip a genuinely higher-scoring impl
below a README"). `rank` reorders in rank space (recall-oriented) and does **not**
preserve the density winner on this adversarial fixture — under RRF the per-shard
density margin has already been compressed to a single rank, so any summary lift
leapfrogs it. That is the honest limit of a rank-domain boost under RRF, and it
is why the fix ships default-inert: a later flip graduates whichever mode the
full-bed bench prefers.

### #265 — local-vs-global IDF trap (`fusion` × `HELIX_SHARD_GLOBAL_IDF`)

| cell | gold rank | wrong rank | gold above wrong? |
|---|---|---|---|
| rrf / gidf=0 | 5 | 2 | no |
| **rrf / gidf=1** | 5 | 2 | no — **byte-identical to gidf=0** ✅ |
| additive / gidf=0 | 1 | 0 | no (the #182 bug) |
| additive / gidf=1 | **0** | 1 | **yes** ✅ |

Reading: under per-shard RRF the flag is a **byte-identical no-op** (guard
holds); under additive the #182 splice still lifts the gold above the wrong doc.
The guard removes a scale-corrupt path, it does not regress the defined one.
